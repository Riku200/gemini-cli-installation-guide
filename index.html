Gemini CLIって、なに？

一言でいうと、「言葉でパソコンを操作できる、魔法の杖」です。

普段、マウスやキーボードで一つずつ行っている作業を、Gemini CLIに「〇〇して」とお願いするだけで、AIが自動で実行してくれます。例えば、チャットで友達に話しかけるような感覚で、複雑な作業も一瞬で終わらせることができるのです。

インストール方法

以下の手順で、あなたも魔法の杖を手に入れることができます。

参考動画を見る (YouTube)

ステップ0: PowerShellを管理者モードで開く

スタートメニュー → PowerShell を右クリック → 「管理者として実行」を選択します。
（Node.js を winget でインストールする際に書き込み権限が必要なため）

ステップ1: Node.js v18以上をインストール

まだNode.jsをインストールしていない場合は、以下のコマンドを実行します。

winget install -e --id OpenJS.NodeJS.LTS

インストール後、バージョンを確認します。

node -v

v18.x.x 以上が表示されればOKです。

ステップ2: Gemini CLIをインストール

次に、Gemini CLI本体をインストールします。これにより、PCのどこからでも `gemini` コマンドが使えるようになります。

npm install -g @google/gemini-cli

ステップ3: 動作確認

`gemini`とだけ入力して起動すれば、セットアップは完了です。

gemini

うまくいかない場合は？

インストール中にエラーが出たり、手順が分からなくなった場合は、問題画面のスクリーンショットを撮影し、GeminiやChatGPTなどのAIアシスタントに「このエラーを解決したい」と質問してみてください。解決策を提示してくれることがあります。

GitHub Pagesでサイトを公開する

作成したサイトを、GitHub Pagesを使ってインターネットに公開する手順です。

前提条件

  • GitがPCにインストールされていること。
  • GitHubのアカウントを持っていること。

ステップ1: GitHubで新しいリポジトリを作成する

まず、公開したいファイルを置くための場所（リポジトリ）をGitHub上に作成します。

  1. GitHubにログインし、ページの右上にある「+」アイコンをクリックして「New repository」を選択します。
  2. Repository name に好きな名前（例: my-awesome-site）を入力します。
  3. Public を選択します。（GitHub PagesはPublicリポジトリなら無料で利用できます）
  4. Create repository ボタンをクリックします。

作成が完了すると、リポジトリのURLが表示されます。このURLは次のステップで使います。

ステップ2: PowerShellでコマンドを実行する

ここからのコマンドをPowerShellに順番にコピー＆ペーストして実行してください。

1. 公開したいファイルがあるフォルダに移動します

cd C:\path\to\your\project

2. Gitリポジトリを初期化します

git init

3. フォルダ内のすべてのファイルをGitの管理対象に追加します

git add .

4. ファイルをコミット（セーブのようなもの）します

git commit -m "Initial commit"

5. デフォルトのブランチ名を「main」に変更します

git branch -M main

6. GitHubで作成したリモートリポジトリを登録します
※下記の <あなたのユーザー名> と <リポジトリ名> をご自身のものに書き換えてください

git remote add origin https://github.com/<あなたのユーザー名>/<リポジトリ名>.git

7. GitHubにファイルをアップロード（プッシュ）します

git push -u origin main

ステップ3: GitHub Pagesを有効にする

最後に、アップロードしたファイルをWebページとして公開する設定を行います。

  1. ステップ1で作成したご自身のGitHubリポジトリのページを開きます。
  2. タブの Settings をクリックします。
  3. 左側のメニューから Pages をクリックします。
  4. Source の項目で、ドロップダウンメニューを Deploy from a branch にします。
  5. Branch の項目で、ブランチを main 、フォルダを /(root) に設定し、Save ボタンをクリックします。

ステップ4: 公開されたページを確認する

設定を保存して数分待つと、ページの上部に "Your site is live at ..." というメッセージと公開用のURLが表示されます。

表示されたURL（ https://<あなたのユーザー名>.github.io/<リポジトリ名>/ ）にアクセスすると、ご自身が作成したWebページが表示されます。

これで公開は完了です！🎉

公開済みサイトの更新方法

サイトの変更は、最初の公開手順ととても似ていて、一度覚えてしまえば簡単です。

サイト更新の基本フロー

サイトの更新は、以下の3つのステップで行います。

  1. パソコン上のファイルを編集する:
    プロジェクトフォルダ内のHTMLやCSSファイルなどを直接編集・保存します。
  2. 変更内容を記録（コミット）する:
    「どこをどう変更したか」という記録をGitに残します。
  3. 記録をGitHubにアップロード（プッシュ）する:
    記録をGitHubに送ると、GitHub Pagesが自動的にサイトを新しい内容に更新してくれます。

具体的な手順（PowerShell）

今後、サイトを更新する際は、以下のコマンドをPowerShellで実行してください。

1. 作業フォルダに移動します

cd C:\path\to\your\project

2. 変更されたすべてのファイルをGitの管理対象に追加します

git add .

3. 変更内容を記録（コミット）します
※ " "の中のメッセージは、何を変更したか分かるように書き換えてください（例: "ヘッダーのテキストを修正"）

git commit -m "サイトの更新"

4. 変更内容をGitHubにアップロード（プッシュ）します

git push

これだけです！git push を実行後、数分待ってからご自身のサイトにアクセスすると、変更が反映されているはずです。

ポイント

ファイルを変更したら、必ずこの手順で git push まで行う必要があります。ローカルファイルを変更しただけでは、公開されているWebサイトは更新されません。

git commit -m "ここ" のメッセージは、後から変更履歴を見返したときに分かりやすいように、具体的な変更内容を書くことをお勧めします。
